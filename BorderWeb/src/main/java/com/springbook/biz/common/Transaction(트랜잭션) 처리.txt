
Transaction : 작업을 처리하는 최소 단위 
	- ALL or Nothing : 전부 되게 하거나 안되면 전부 안되게 처리.
	- 
	- Insert, Update, Delete 구문에서 트랜잭션이 발생
	- Oracle DB : Insert, Update, Delete 구문 앞에 begin Transaction이 자동으로 등록, commit; 명시
	- MSSQL, MySQL : 자동 commit;
	- Connection 객체는 기본셋이 자동 commit;
	- commit; 전까지는 메모리에서만 적용, commit; 이후에는 DB에 저장.
	- 모든 정보가 log에 기록이 됨. Transaction log,
	- 백업된 시점이 아니라, 오류가 발생된 시점 복구
	- 모든 Insert, Update, Delete 구문 내에서 트랜잭션 처리
	
	
	- Spring의 트랜잭션 처리는 AOP를 통해서 처리, XML설정에서만 설정이 가능, 어노테이션 사용할 수 없다.
	- Spring에서 트랜잭션 처리를 하기 위해서는 트랜잭션 관리자를 생성해야 한다.
	- Spring에서 트랜잭션 처리는 메소드 단위로 처리가 됨.
	
	- 예) A 통장에서 B 통장으로 1억을 입금. 두 Update구문은 하나의 트랜잭션으로 처리 되어야 한다.
		update A - 1억
		update B + 1억
